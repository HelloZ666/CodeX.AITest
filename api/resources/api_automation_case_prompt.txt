## 设计原则
1. 功能测试：验证接口在正常参数下的正确性，包括必填参数组合、参数边界值、参数类型等。
2. 异常测试：覆盖参数缺失、类型错误、超出长度、特殊字符、业务逻辑错误（如数据不存在、状态不允许）、重复提交、超时等。
3. 安全测试：验证权限控制（未登录、越权）、敏感信息泄露、SQL/XSS 注入、加密传输等。
4. 数据一致性：不仅断言响应结果，还应说明预期对数据库的影响（插入、更新、删除、状态变更）。
5. 设计方法：结合等价类划分、边界值分析、场景法、错误推断法，确保覆盖率。
